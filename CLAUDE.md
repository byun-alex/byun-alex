# Working in Alex's GitHub repos

## Writing style (important)
- **No em-dashes or long dashes (- / -).** They read as AI-written and undercut the portfolio. Use commas, colons, parentheses, or short separate sentences instead. A normal short hyphen in a list item (`name - description`) is fine.
- Keep the voice plain, concrete, and human. Match the existing `README.md` / `PORTFOLIO.md`.
- Proof over hype: describe what a thing does and how it's built, not adjectives.
